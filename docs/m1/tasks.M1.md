## 阶段五：体验优化（Bug Fixes）
- [x] 5.1 备选区三个积木贴得过近导致视觉重叠
  - **描述**：备选区内相邻的三个积木水平间距过小，贴合处出现色块重叠，影响可读性与可点击/拖拽命中。
  - **解决方案（方案B 更优）**：
    - 使用等距网格布局：容器改用 `grid`，拉开间距。
    - 增加容器宽度：以确保 3x3 积木有充足空间不被挤压。
    - 增加视觉区分：为积木添加 1px 半透明描边与外阴影，降低相邻色块贴合感。
    - 增加交互反馈：拖拽 hover/focus 时增加外发光与底色，避免视觉融合。
- [x] 5.2 L 形积木拖拽时出现 2x2/3x3 外框阴影
  - **描述**：拖拽 L 形积木时，浏览器默认拖拽快照会带出外接正方形阴影，影响视觉准确性。
  - **解决方案**：
    - 移除桌面端原生 HTML5 Drag 预览，统一改为 Pointer 事件驱动的拖拽流程。
    - 保留棋盘合法落点预览作为唯一拖拽反馈，避免外接正方形阴影误导。
- [x] 5.3 修复拖拽出现“圆球”而非积木跟随的问题
  - **描述**：在部分环境下，拖拽会出现异常系统拖影（圆球样式），导致交互体验错误。
  - **解决方案**：
    - 完全取消原生拖拽依赖，桌面与移动端统一使用 Pointer 流程。
    - 通过 `elementFromPoint` 命中网格单元并实时更新落点预览，保证表现一致。
