# 里程碑 M3｜ThousandTen 当前版本完整规格说明书

## 1. 产品定位
- **游戏类型**：单机放置消除类休闲游戏（Endless）。
- **核心体验**：规则稳定可控，操作直觉清晰，反馈爽快且连续。
- **当前版本目标**：在 M2 完整玩法基础上，新增 M3 反馈系统能力与移动端触控可视性优化。

## 2. 游戏场地与回合机制
- **场地尺寸**：10x10 网格。
- **备选区补货规则**：每轮固定生成 3 个备选积木，必须全部使用后才会刷新下一批。
- **放置顺序**：玩家可自由选择 3 个备选积木的放置顺序。
- **结束判定**：当备选区所有剩余积木都无法放入棋盘任何合法位置时，游戏结束。
- **重开机制**：支持一键重开，重置棋盘、分数、备选区及反馈状态。

## 3. 积木体系
- **基础尺寸约束**：常规积木最大不超过 3x3。
- **基础积木类型**：
  - 1x1 点块
  - 1x2 / 1x3 直线块（横向、纵向）
  - 2x2 / 3x3 正方形块
  - L 形块（2x2 与 3x3 变体）
- **扩展积木类型**：
  - 1x4 长条块（横向、纵向）
  - 1x5 长条块（横向、纵向）
- **生成方式**：每轮从积木池随机抽取 3 个，默认采用均衡随机策略。

## 4. 放置、消除与计分
- **放置合法性**：积木不可越界、不可与已占用格重叠。
- **消除规则**：任意整行或整列填满时触发消除，支持多行、多列与行列交叉同时消除。
- **基础得分**：成功放置积木得 `n` 分（`n` 为积木占用格数）。
- **消除奖励**：每消除 1 行或 1 列，额外 `+10` 分。
- **连击约束**：连击仅增强反馈表现，不增加额外积分，不改变基础计分模型。

## 5. 交互系统
- **拖拽交互**：支持鼠标与触控拖拽；拖拽过程提供实时落点预览。
- **预览反馈**：合法落点与非法落点使用明确视觉状态区分。
- **拖拽贴合**：拖拽浮层与指针位置保持高一致性，减少偏移与跳变。
- **触控可视性优化（M3）**：触控拖拽时积木浮层相对手指上移，降低手指遮挡；鼠标拖拽保持原有对齐。
- **命中容错**：积木抓取区域具备可用的交互容错，降低“点空”概率。

## 6. 反馈系统
### 6.1 视觉反馈
- 放置、消除、补货具备统一节奏与动效语言。
- 连击场景提供逐级增强的视觉提示，断连后正确重置层级。
- 高强度反馈下仍保证棋盘信息可读，不遮挡关键落点判断区域。

### 6.2 音效反馈（M3）
- 提供落子音、消除音、连击音阶。
- 连击时音效层级随反馈增强，形成连续爽感。
- 提供音效开关与音量档位，支持本地持久化。

### 6.3 触觉反馈（M3）
- 支持振动设备在落子、消除、连击时触发差异化短振动。
- 不支持振动设备自动降级，不影响核心玩法。
- 提供振动开关，关闭后不再触发触觉反馈。

## 7. 模式范围与非目标
- **当前模式范围**：仅保留单机 Endless 主玩法。
- **当前版本不包含**：关卡系统、局内变体规则、技能或道具、成长养成、社交分享、排行榜、商业化模块。

## 8. 技术与实现约束
- **技术栈**：React + Vite + TypeScript + Tailwind CSS。
- **架构要求**：核心规则逻辑与 UI 展示层分离，核心算法可独立测试。
- **状态管理**：网格状态保持不可变更新，核心状态集中管理，避免多源不一致。
- **关键判定时机**：Game Over 必须在“成功放置后”和“备选补货后”两处执行。
- **一致性要求**：新增反馈能力不得改变既有规则判定与计分结果。
