# 里程碑 M2｜10x10 积木消除游戏 (ThousandTen) 规格说明书

## 1. 游戏概述
一款基于 10x10 网格的放置消除类休闲游戏。玩家每轮从备选区获得 3 个随机积木，自由选择顺序进行放置；填满整行或整列可触发消除并获取额外分数。当备选区积木都无法合法放置时游戏结束。M2 目标是在扩展棋盘与积木种类的同时，提升交互流畅度与整体视觉品质。

## 2. 核心规则
- **场地尺寸**：10x10 方格。
- **积木生成**：每次在备选区随机生成 3 个积木，3 个全部放置后刷新下一批。
- **放置规则**：积木可自由选择顺序放置；不可重叠，不可越界。
- **消除规则**：任意整行或整列被填满时触发消除，支持多行多列同时消除。
- **结束条件**：当备选区任意剩余积木都无法放入棋盘任意合法位置时，判定 Game Over。

## 3. 计分机制
- **基础放置分**：放置积木获得 `n` 分（`n` 为积木占用格子数）。
- **消除奖励分**：每消除 1 行或 1 列，额外 `+10` 分。
- **多消示例**：同时消除 1 行 1 列得 `+20`；同时消除 2 行 1 列得 `+30`。
- **总分计算**：放置分 + 消除奖励分。

## 4. 积木体系
- **尺寸约束**：基础积木最大不超过 3x3。
- **基础积木类型**：
  - 1x1 点块
  - 1x2 / 1x3 直线块（横向、纵向）
  - 2x2 / 3x3 正方形块
  - L 形块（2x2 与 3x3 变体）
- **M2 新增积木**：
  - 1x4 长条块（横向、纵向）
  - 1x5 长条块（横向、纵向）
- **生成策略**：每轮从积木池随机生成 3 个积木；生成权重可配置，默认保持均衡体验。

## 5. 交互与视觉体验目标（M2）
- **拖拽平滑性**：拖拽预览与落点高亮同步更新，避免跳帧与闪烁。
- **动效统一性**：放置、消除、补货三类动效采用统一节奏与缓动曲线。
- **可读性提升**：10x10 小格尺寸下，保证网格线、积木边界、预览状态清晰可辨。
- **响应式一致性**：移动端与桌面端交互反馈一致，保持触控与鼠标体验同构。

## 6. 技术约束与实现边界
- 前端框架：React + Vite + TypeScript + Tailwind CSS。
- 游戏逻辑继续保持“逻辑层与展示层分离”。
- 网格状态更新必须保持不可变数据结构。
- Game Over 判定必须在“成功放置后”和“补货后”两处精确执行。
